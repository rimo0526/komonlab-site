## Stack

| Layer            | Choice                                  |
| ---------------- | --------------------------------------- |
| Framework        | [Astro 4.x](https://astro.build) (static output) |
| Styling          | Tailwind CSS 3 (custom theme)           |
| Interactive UI   | Preact island (only where required)     |
| PDF generation   | `pdf-lib` (client-side)                 |
| Sitemap          | `@astrojs/sitemap` (pinned `3.2.1` for Astro 4 compatibility) |
| Type checking    | `@astrojs/check` + TypeScript strict    |
| Fonts            | Google Fonts CDN вЂ” Fraunces, Inter (both OFL) |
| Hosting (planned)| Cloudflare Pages                        |
| Analytics (planned) | Umami (self-hosted or shared)        |

No third-party JS analytics, no signup, no tracking.

## Quick start

```bash
# 1. Install dependencies
npm install

# 2. Start the dev server (http://localhost:4321)
npm run dev

# 3. Type-check + diagnostics
npm run check

# 4. Production build в†’ dist/
npm run build

# 5. Preview the production build locally
npm run preview
```

### Note about `node_modules`

The session that scaffolded this project couldn't keep `node_modules`
inside the project directory (sandbox mount restriction). The first
thing you should do on a fresh checkout is:

```bash
rm -rf node_modules package-lock.json
npm install
```

A `package-lock.json` will be generated by the first install.

## Design system summary

Full source lives in `_work/tool-first-export-project/design-system.md`.
The Tailwind config implements it. Quick reference:

### Color tokens

| Token              | Hex       | Use                                   |
| ------------------ | --------- | ------------------------------------- |
| `primary-700`      | `#1F3A5F` | и—Ќ (ai indigo) вЂ” primary action, headings |
| `secondary-500`    | `#8C3A2A` | ејЃжџ„ (bengara) вЂ” eyebrows, accents     |
| `accent-500`       | `#6B7F3F` | жЉ№иЊ¶ (matcha) вЂ” sparingly              |
| `neutral-50`       | `#FAF6EE` | Page background                       |
| `neutral-700`      | `#3A352A` | Body text                             |
| `success / warning / error / info -500` | вЂ” | WCAG AA semantic colors |

A full 50вЂ“900 scale is provided for primary, secondary, accent, and neutral.

### Typography

| Role    | Family                | Notes                       |
| ------- | --------------------- | --------------------------- |
| Display | Fraunces (Variable)   | Headings, logo wordmark     |
| Body    | Inter (Variable)      | Default body                |

Modular scale 1.250 (`text-xs` в†’ `text-5xl`).

### Layout

- `max-w-container` = 1200px
- `max-w-prose` = 65ch
- Container utility: `.kl-container`

### Accessibility

- WCAG 2.1 AA color contrast verified at design-system stage.
- Skip-to-content link in `BaseLayout`.
- `:focus-visible` ring globally via `global.css`.
- `prefers-reduced-motion` honored.
- Semantic `<header>`, `<nav aria-label="вЂ¦">`, `<main>`, `<footer>`.

## Deployment (planned вЂ” Phase 2)

Target: **Cloudflare Pages**.

1. Connect this repository (or upload the `dist/` directory) to Cloudflare Pages.
2. Build command: `npm run build`
3. Build output directory: `dist`
4. Environment: `NODE_VERSION=22` (or `>=18.17`).
5. Once a custom domain is acquired, update `site` in `astro.config.mjs`.

No deployment is performed by this project step вЂ” the build is verified
locally only.
